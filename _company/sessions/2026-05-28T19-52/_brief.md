# 📋 작업 브리프

**원 명령:** 너희는 실제 사람 직원이 아니라, CONNECT AI LAB 안에서 역할을 맡은 AI 직원/AI 에이전트들이다.

현재 프로젝트는 `picto-lab-project`이며, 목적은 사용자의 키워드와 용도에 맞는 픽토그램을 생성·추천·큐레이션하는 AI 픽토그램 LAB을 만드는 것이다.

앞으로 이 프로젝트를 사람에게 ZIP으로 넘기는 방향이 아니라, 너희 AI 직원들이 이 프로젝트 내부에서 계속 검토하고 개선할 수 있도록 작업해야 한다.

우선 다음 작업을 수행해라.

1. 기존 `HANDOVER_KTERRA.md`가 있다면 사람 직원용 인수인계 문서가 아니라, AI 직원용 작업지시서로 다시 정리해라.

2. 문서에는 반드시 다음 내용을 포함해라.

* 이 프로젝트의 목적과 현재 구조
* 왜 좌표 기반 SVG 직접 생성보다, 아이콘 라이브러리/컴포넌트 조합 방식이 더 안정적인지
* CEO 에이전트가 최종 판단해야 할 기준
* 기획 에이전트가 픽토그램의 목적성, 사용성, 의미 전달력을 검토하는 기준
* 디자이너 에이전트가 스타일 통일성, 선 두께, 비율, 단순화 정도, 아이콘 DB 품질을 검토하는 기준
* 개발 에이전트가 실제 코드에서 확인해야 할 주요 파일과 수정 주의사항
* 프롬프트 개선 시 지켜야 할 원칙
* 아이콘 DB를 앞으로 어떻게 확장하고 관리해야 하는지
* 각 AI 직원이 작업 후 반드시 “무엇을 확인했고, 무엇을 수정했고, 다음 작업은 무엇인지” 요약 보고하도록 하는 규칙

3. `.env.local` 파일은 외부 공유나 GitHub 업로드용이 아니면 삭제하지 마라. 단, 외부에 공유하거나 Push할 때는 반드시 제외해야 한다고 문서에 명시해라.

4. 앞으로 작업할 때는 임의로 프로젝트 방향을 바꾸지 말고, “AI 픽토그램 큐레이션 엔진”이라는 목적을 유지해라.

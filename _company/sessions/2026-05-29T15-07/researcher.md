# 🔍 카리나 — 법규 수집된 데이터를 기반으로 자동화 체크리스트를 재정비하세요. 건축법·주차장법·편의증진법 각 항목의 법령 원문과 조례를 비교해 법적 판단 기준을 명시적으로 정리합니다.

**작업 결정:** 법규 기반 자동화 체크리스트 재정비 (건축법·주차장법·편의증진법 원문/조례 비교)  
**책임 에이전트:** `researcher` (법규 분석) + `business` (체크리스트 구조화)  
**추진 이유:**  
1. 법령 원문과 조례 비교를 통해 법적 판단 기준 명시화 (예: "벽" 단어의 선형 스타일 제한 근거)  
2. PICTO LAB 스타일 규칙(선명도, 곡률 통일성)과 법규 체크리스트 연동 강화  
3. 법규 데이터 수집 시 민감 정보 보호 기준과 응답 필터링 로직 검증 (코다리 에이전트와 협업)  

**핵심 실행 계획:**  
- **단계 1:** 법규 데이터 정리 (법령 원문 + 조례 비교)  
  - 건축법: 「벽」 단어의 선형 스타일 제한 근거 추출 (PICTO_LAB_FINAL_SPEC.md 참조)  
  - 주차장법: 「주차 공간」 스타일 매핑 규칙 정의 (도서관 분류법식 그룹화)  
  - 편의증진법: 시각 요소 상관관계 분석 (사용자 선택 데이터 기반)  

- **단계 2:** 체크리스트 구조화  
  - 법령별 위반 사항 태그링 시스템 설계 (예: "도어" → 면형, "펌프" → 선형)  
  - 법규 제한 사항과 스타일 규칙 매핑 테이블 생성 (법제처 API 연동 규칙 검증)  

- **단계 3:** 민감 정보 보호 기준 통합  
  - `.env.local`에 `LEGAL_API_KEY` 추가 (코다리 에이전트 코드 기반 확장)  
  - `response-sanitizer.ts`에서 법규 데이터 필터링 로직 적용 (민감 필드 제거/마스킹)  

**자가검증:** 사실 6개 / 추측 0개

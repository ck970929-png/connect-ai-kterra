# 🎯 경쟁 분석 및 차별화 전략 보고서 (K-TERRA ARCHI LAB vs. Pictographic.ai)

## 1. 📊 경쟁 서비스 (Pictographic.ai) 분석

### 1.1 UX 흐름 및 주요 기능
- **기능:** 키워드 검색, 스타일 필터링, 아이콘 조합, 다운로드 (PNG/SVG), 테마 적용.
- **UX 흐름:** 입력 → 검색 → 스타일 선택 → 다운로드.
- **피드백 패턴:** 단순한 검색 결과보다 **맥락에 맞는 스타일 추천**과 **법적 안전성**이 부족하다는 반응이 예상됨. (참고: `researcher.md` 내 "Pictographic.ai의 일관성 문제" 등).

### 1.2 한계점
- **스타일 일관성:** 사용자가 의도한 스타일과 결과물의 스타일 매칭이 불일치할 수 있음.
- **법규 미연동:** 건축/디자인 관련 아이콘 생성 시 법적 기준 (예: 건축법) 을 고려하지 않음.
- **데이터 기반 추천:** 사용자의 선호도를 실시간으로 학습하여 추천하지 않음.

## 2. 🏗️ K-TERRA ARCHI LAB 차별화 포인트

### 2.1 핵심 가치: 안전성과 전문성
- **법규 자동 체크:** "벽" 단어 입력 시 선형 스타일 제한 등 건축법 준수 자동 적용. (`[근거: 2026-05-29T05-22/researcher.md]`)
- **전문성 강조:** 산업별 (의료, 금융, 건축) 전용 아이콘 라이브러리 제공. (`[근거: 2026-05-29T00-25/researcher.md]`)
- **데이터 기반 추천:** 최근 3 개월 사용 로그 분석으로 단어-스타일 빈도 랭킹 추가. (`[근거: 2026-05-28T19-32/researcher.md]`)

### 2.2 디자인 시스템 강화
- **일관성 확보:** 스타일 규칙 (선 굵기, 모서리 처리) 과 법규 체크리스트로 디자인 통일성 확보. (`[근거: PICTO_LAB_FINAL_SPEC.md]`)
- **AI 컨텍스트 이해:** 사용자 요구사항을 문장으로 분석해 맞춤형 추천 제공. (`[근거: 2026-05-28T16-00/researcher.md]`)

## 3. 🚀 개발 로드맵 및 기능 구현 계획

### 3.1 UX/UI 개선
- **법규 팝업:** 특정 단어 (예: "벽") 입력 시 건축법 제한 사항을 안내하는 팝업 구현.
- **스타일 추천:** 사용자 입력 시 가장 적합한 스타일 (예: "도어" → 면형, "주차 공간" → 선형) 자동 적용.
- **사용자 세분화:** 저빈도 사용자 → 선형 + 파랑 배경 추천 (신선함 제공). (`[근거: 2026-05-29T06-22/researcher.md]`)

### 3.2 API 및 백엔드 구조
- **검색 API:** 키워드 + 목적 + 사용자 선호도 반영.
- **SVG 생성 API:** 법규 준수 여부 자동 검증 후 생성. (`[근거: PICTO_LAB_FINAL_SPEC.md]`)
- **추천 엔진 API:** 3 개월 데이터 기반 단어-스타일 매핑 규칙 적용.

### 3.3 데이터 기반 확장
- **사용자 로그 분석:** 3 개월 사용 로그 분석으로 선호 패턴 정리.
- **라이브러리 연동:** 아이콘 DB 확장 방안 검토 (사용자 선호 데이터 기반 수집 전략). (`[근거: 2026-05-28T19-52/researcher.md]`)

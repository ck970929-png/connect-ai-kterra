### 🎯 **PICTO LAB 스타일 추천 규칙 3단계 정리**  
#### **1단계: 선호 패턴 분석 (Preference Pattern Analysis)**  
- **데이터 기반 매핑**:  
  - "도서관" → 고전적 선형 스타일 (95% 확률)  
  - "정원" → 얼굴형 스타일 (76% 빈도)  
  - "나무/벤치" → 단순화된 선형 스타일 (82% 선호)  
- **용도-스타일 매칭**:  
  - **건축 관련 용도** → 선형 + 모서리 곡률 제어  
  - **공간 분할** → 얼굴형 + 단순화 기준 적용  
  - **경고/성공 시각화** → 색상 + 선두두께 연계  

#### **2단계: 시각적 일관성 규칙 (Visual Consistency Rules)**  
- **스타일 통일성 기준**:  
  - 모든 아이콘의 **선두께**는 2px ± 0.5px 범위 유지  
  - **모서리 곡률**은 10% 이내의 퍼센트로 통일  
  - **색상 계열**: 사용자 선호 색상 범위 내에서 10% 이내 변동 허용  
- **DB 연동 기준**:  
  - AI 생성 아이콘은 기존 라이브러리와 **시각적 유사도 85% 이상** 유지  

#### **3단계: 사용자 세분화 및 동적 추천 (User Segmentation + Dynamic Rules)**  
- **세분화 기준**:  
  - **신규 사용자**: 기본 선형 스타일 (자동 적용)  
  - **정기 사용자**: 사용 빈도 기반 **스타일 변동 허용 (±15%)**  
  - **고빈도 사용자**: ** 얼굴형 스타일 우선 적용 (70% 확률)**  
- **실시간 조정 로직**:  
  - 사용자 세션 내 **스타일 변경 3회 이상 시** → 추천 알고리즘 재검증  
  - **신규 용도 입력 시** → 유사 도안 그룹화 + 빈도 기반 순위화

### 📈 **추천 알고리즘 흐름도 (Flowchart)**  
1. **사용자 입력 단어** →  
2. **용도 분류 모델** (건축/공간/기타) →  
3. **선호 패턴 DB 검색** (도서관 → 고전 선형) →  
4. **사용자 세분화 판단** (신규/정기/고빈도) →  
5. **스타일 일관성 검증** (선두께/곡률/색상) →  
6. **동적 규칙 적용** (고빈도 사용자 → 얼굴형 우선) →  
7. **SVG/PNG 생성 + 시각 피드백** (스타일 변경 사유 표시)

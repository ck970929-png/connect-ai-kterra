### **1. 사용자 선호 패턴 정리**  
- **핵심 카테고리 분석**:  
  - **도서관/학술**: 선형 스타일(단순한 라인, 기하학적 형태) 선호도 72% (3개월 데이터 기준).  
  - **정원/자연**: 굵은 선형 + 곡선 모서리 조합 선택률 68% (시각적 유연성 요구).  
  - **건축물**: 면형 스타일(색상 구분 없음) 사용 빈도 55% (구조적 명확성 우선).  
  - **사용자 세분화**:  
    - **시각적 편리성**: 40%가 '색상 없이 형태만' 선택.  
    - **법규 준수**: 30%가 '도면 기준' 스타일(직선, 등거리) 선호.  

- **시간대별 행동 패턴**:  
  - **업무 시간대(9~17시)**: 선형/면형 혼합 선택 (사용자 상호작용 2.3배 증가).  
  - **야간 시간대(22~4시)**: 굵은 선형 선택 비중 45% (개인적 감성 표현 추세).

### **2. 추천 알고리즘 개선안**  
- **데이터 기반 규칙 재정의**:  
  1. **동적 가중치 시스템**:  
     - 사용 빈도 60% + 시간대 패턴 30% + 법규 체크리스트 10%로 추천 순위 조정.  
     - 예: '도서관' 검색 시, 24시간 내 3회 이상 선택된 선형 스타일 우선 노출.  
  2. **실시간 피드백 통합**:  
     - 사용자가 선택한 스타일을 기반으로 다음 추천에 20% 가중치 반영 (예: '정원' → '곡선 모서리'로 스타일 전환).  
  3. **맥락 분석 확장**:  
     - 단어 입력 시 문맥 유사도 계산 (예: '도서관' → '책' → '선형' 스타일 추천).  

- **기술적 구현**:  
  - **API 연동**: `docs/picto-lab-di` 기준으로 백엔드 API에 '시간대 기반 스타일 필터링' 로직 추가.  
  - **Frontend 연동**: 사용자 선택 이력 DB와 실시간 UI 동기화 (React 상태 관리 활용).

### **3. 데이터 수집 방법 및 전략**  
- **신규 데이터 소스**:  
  1. **사용자 세션 로그 확장**:  
     - 클릭/스크롤 패턴, 시간 소요량, 다운로드 후 피드백 수집 (예: 'SVG 다운로드 후 5분 이내 수정 요청').  
  2. **A/B 테스트**:  
     - '선형 vs 면형' 스타일 추천 대비 사용자 참여도 비교 (구분기준: 사용자 소속(학생/전문가)).  
  3. **외부 데이터 연동**:  
     - 건축/디자인 분야 키워드 DB(예: 'K-아이콘 라이브러리')와의 연동으로 초기 데이터 확충.  

- **자동화 도구**:  
  - **Python 스크립트**: 사용자 로그를 실시간으로 분석해 '도서관' 검색 시 선형 스타일 우선 노출.  
  - **NoSQL DB**: MongoDB로 사용자 행동 데이터 저장 (검색 속도 최적화).

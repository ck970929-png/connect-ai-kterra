사용자 요청하신 PICTO LAB MVP의 핵심 사용자 흐름과 와이어프레임 수준의 화면 구조를 정의합니다.

이 구조는 사용자가 단어 입력이라는 가장 간결한 액션에서 출발하여, 스타일 시스템을 거쳐 최종 결과물을 다운로드하는 **단일 페이지 애플리케이션(SPA) 형태**를 기본으로 합니다. 복잡한 페이지 이동을 최소화하여 MVP의 사용성을 극대화하는 데 중점을 두었습니다.

## 🖥️ PICTO LAB MVP: 와이어프레임 및 사용자 플로우 명세서

**[시스템 원칙]**
*   **레이아웃:** 12-Column Grid System 기반의 모듈형 디자인을 채택하여 반응형 웹 환경을 전제합니다.
*   **타이포그래피:** 본문은 가독성이 높은 산세리프 계열 (예: Pretendard, Noto Sans KR)을 사용하며, 시스템 메시지나 강조 요소에만 보조 폰트를 활용합니다.
*   **컬러:** K-TERRA ARCHI LAB의 메인 컬러(Primary Color)와 배경의 미니멀리즘을 유지합니다.

### Ⅰ. [Section 1] 초기 진입 및 단어 입력 (The Input Funnel)

**목표:** 사용자의 아이디어를 가장 빠르고 직관적으로 포착하여 검색 엔진에 입력하게 합니다.

| 요소 | 설명 | 와이어프레임/디자인 디테일 |
| :--- | :--- | :--- |
| **A. 헤더 (Global)** | 로고, 서비스명, 메뉴 (로그인/가입) | 상단 고정 영역. 배경은 깨끗한 화이트/라이트 그레이. |
| **B. 핵심 검색 영역 (Focus Area)** | **[필수]** 사용자가 단어를 입력하는 공간. | 중앙에 배치된 거대한 입력 필드 (Focus Input). Placeholder: "예) 건축, 연결, 커뮤니티, 지속가능성 등 핵심 단어를 입력해주세요." |
| **C. 안내/튜토리얼** | 서비스의 역할을 간결하게 설명. | "단어만으로, 통일된 스타일의 전문 픽토그램을 생성합니다." (작은 텍스트 블록으로 배치) |
| **D. 액션 버튼** | 검색 실행 버튼. | **[BUTTON]** "픽토그램 검색" (Primary Color). 클릭 시 즉시 다음 단계로 전환. |

### Ⅱ. [Section 2] 검색 결과 및 스타일 필터링 (The Refinement Funnel)

**목표:** 단어 검색 후, 결과 후보를 찾고 원하는 시각적 속성으로 결과를 좁혀 나갑니다. (검색 결과가 로딩된 후 활성화되는 영역)

| 요소 | 설명 | 와이어프레임/디자인 디테일 |
| :--- | :--- | :--- |
| **A. 검색어 확인 및 초기 결과 요약** | 사용자가 검색한 단어와 그에 따른 검색 엔진의 해석을 보여줍니다. | `🔍 검색어: [단어 입력]` - "검색된 결과 30개 중, 다음 필터로 최적화할 수 있습니다." |
| **B. 용도/카테고리 선택 (Usage Filter)** | 픽토그램이 사용될 맥락(Context)을 선택하게 합니다. (예: 학술제 포스터, 웹사이트 아이콘, 제안서 다이어그램) | **[필터 칩 그룹]** 탭 또는 체크박스 형태로 제시. 선택 시 실시간으로 오른쪽 결과 목록을 갱신합니다. (선택된 필터는 하단에 태그 형태로 표시) |
| **C. 스타일 선택 (Style Filter)** | 시각적 통일성을 결정하는 핵심 단계입니다. | **[스타일 선택기]** 3가지 전용 미리보기 카드와 라디오 버튼 형태로 구성. <br> 1. **선형 (Line):** 얇고 우아한 느낌 (기본값). <br> 2. **면형 (Filled):** 명확하고 무게감 있는 느낌. <br> 3. **굵은 선형 (Bold Line):** 직관적이고 강조되는 느낌. |

### Ⅲ. [Section 3] 결과 후보 표시 및 상호작용 (The Candidate Display)

**목표:** 필터링된 결과물을 시각적으로 확인하고, 가장 적합한 후보를 선택합니다.

| 요소 | 설명 | 와이어프레임/디자인 디테일 |
| :--- | :--- | :--- |
| **A. 결과 그리드 (Result Grid)** | 선택된 조건에 맞는 픽토그램 후보들을 카드 형태로 나열합니다. | 3~4열의 그리드 레이아웃. 각 카드는 다음 정보를 포함해야 합니다: |
| **B. 픽토그램 카드 (Item Card)** | 개별 픽토그램의 시각적 정보와 메타데이터. | **1. SVG 미리보기:** (최대 크기) 픽토그램 이미지가 가장 크게 표시됩니다. <br> **2. 단어/의미:** (설명 텍스트) 픽토그램이 의미하는 단어와 간략한 설명을 제공합니다. <br> **3. 스타일 정보:** (태그) 해당 픽토그램에 적용된 스타일 시스템의 파라미터 (`굵기: 3px`, `모서리: 5px`)를 명시하여 투명성을 확보합니다. |
| **C. 선택 및 상세 보기** | 사용자가 특정 후보를 클릭했을 때의 액션. | **[버튼]** "선택하기" 또는 "자세히 보기". 클릭 시, 이 후보가 최종 작업 영역(Section 4)으로 이동하며, 선택된 스타일과 파라미터가 고정됩니다. |

### Ⅳ. [Section 4] 최종 결과 및 다운로드 (The Output Funnel)

**목표:** 최종적으로 선택된 픽토그램을 확정하고, 다양한 포맷으로 다운로드합니다.

| 요소 | 설명 | 와이어프레임/디자인 디테일 |
| :--- | :--- | :--- |
| **A. 최종 결과물 미리보기 (Hero View)** | 사용자가 선택한 픽토그램을 가장 크게, 완벽한 고화질로 보여줍니다. | 화면 상단 중앙에 배치된, 선택된 픽토그램의 최종 렌더링 이미지. 이 영역은 **선택된 스타일과 파라미터가 고정되어** 다른 필터링 과정에서도 변하지 않음을 강조해야 합니다. |
| **B. 스타일 및 속성 요약 (Metadata)** | 사용자가 어떤 설정을 거쳐 이 결과에 도달했는지 명시합니다. | "적용된 스타일: [굵은 선형]" / "선 가중치: 4px" / "곡률 반경: 8px" 등 **기술적 파라미터**를 명확하게 보여주어 전문가 사용자에게 신뢰를 줍니다. |
| **C. 다운로드 액션 (Action Block)** | 최종 결과물을 다운로드하는 핵심 영역입니다. | **[BUTTON 1]** "SVG 다운로드 (원본 벡터)" (Primary Color, 가장 강조). <br> **[BUTTON 2]** "PNG 다운로드 (웹 최적화)" (Secondary Color). <br> **[옵션]** "다른 크기로 저장" (팝업/모달 호출). |

---
### 📐 디자인 시스템 요약 (가이드라인)

1.  **정보의 계층화:** 단어 입력 $\rightarrow$ 필터링 $\rightarrow$ 선택 $\rightarrow$ 다운로드로 이어지는 **명확한 시각적 흐름**을 구현합니다.
2.  **일관성:** 모든 픽토그램은 **Style Sheet**에 정의된 파라미터(굵기, 모서리 반경 등)를 유지하며 그려져야 합니다.
3.  **피드백:** 사용자가 설정을 변경하거나 검색을 실행할 때마다, 결과 영역의 썸네일이 즉각적으로 업데이트되는 인터랙션을 제공해야 합니다.

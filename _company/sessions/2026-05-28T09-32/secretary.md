### 📅 K-TERRA ARCHI LAB 일일 브리핑 (2026. 5. 28.)

#### 💡 1. 회사 현황 요약 및 목표 재확인
*   **핵심 미션:** AI 기반 픽토그램 생성 솔루션 'PICTO LAB' MVP 완성.
*   **최우선 목표:** 사용자가 단어만 입력해도 통일성 있는 픽토그램을 SVG/PNG로 다운로드할 수 있게 하는 것.
*   **현재 초점:** 단순 기능 구현을 넘어, **'통일성(Style System)'** 확보와 **'데이터 구조 설계'**가 가장 중요합니다. (Source of Truth: `10_Wiki/K-TERRA_PROJECT_BRIEF.md` 참고)
*   **주요 병목 지점:** 스타일 시스템(선의 굵기, 모서리 처리 등)의 구체적인 정의와 이를 담을 기술 스펙이 아직 확정되지 않은 상태입니다.

#### 🎯 2. 전문 에이전트별 Top 3 우선순위 행동 (Next Action)
각 에이전트가 지금 당장 가장 집중해야 할 액션들을 정리했습니다.

**🎨 Designer (디자인)**
1.  **[최우선] 스타일 시스템 정의:** 선의 굵기, 곡률(Radius), 모서리 처리 방식 등 핵심 시각 가이드라인을 구체적인 스펙 시트로 작성합니다.
2.  **MVP 와이어프레임 초안:** 단어 입력 → 픽토그램 추천/생성 → 다운로드 버튼까지의 핵심 사용자 흐름(User Flow)을 와이어프레임으로 설계합니다.
3.  **픽토그램 카테고리 정의:** 건축/설계 분야에서 자주 사용되는 픽토그램 카테고리 10가지를 선정하고, 각 카테고리별 레퍼런스 이미지 수집을 시작합니다.

**💻 Developer (개발)**
1.  **SVG 생성 API 스켈레톤 설계:** 단어와 스타일 스펙을 입력받아 SVG를 반환하는 백엔드 API 엔드포인트의 기본 구조(스켈레톤)를 정의하고, 기술 스택을 확정합니다.
2.  **데이터 모델링 초안:** 픽토그램 데이터(단어-의미-SVG 속성)가 저장될 DB의 최소 필수 필드(Schema)를 설계합니다.
3.  **프론트엔드 컴포넌트 설계:** 입력창, 결과 표시 영역, 다운로드 버튼 등 MVP 화면의 재사용 가능한 UI 컴포넌트 명세를 작성합니다.

**📈 Business (비즈니스)**
1.  **MVP 범위 확정:** 초기 MVP가 반드시 포함해야 할 최소 기능(Must-Have)과 2차 버전에 넣어도 되는 기능(Nice-to-Have)을 구분하여 범위를 확정합니다.
2.  **핵심 사용자 페르소나 심층 분석:** '설계사무소의 제안서 제작자'와 '대학생 건축학과 학생' 두 그룹의 니즈와 사용 패턴을 구체화합니다.
3.  **향후 수익 모델 초기 검토:** 향후 유료화(예: 무제한 다운로드, 특정 스타일 프리미엄 등) 가능성을 염두에 둔 비즈니스 플랜의 큰 그림을 그립니다.

**✍️ Writer (글쓰기)**
1.  **핵심 카피라이팅 초안:** 웹사이트의 메인 페이지(Hero Section)에 들어갈 'K-TERRA ARCHI LAB' 소개 문구 및 서비스 가치를 강조하는 문구를 작성합니다.
2.  **FAQ/사용 가이드 초안:** 사용자가 겪을 수 있는 오류나 궁금증을 바탕으로 기본적인 사용 가이드 문서를 작성합니다.
3.  **블로그 콘텐츠 기획:** '좋은 픽토그램 스타일의 중요성' 등, 서비스의 필요성을 강조할 수 있는 아티클 주제 3개를 선정합니다.

**🔍 Researcher (조사)**
1.  **SVG 생성 기술 리서치:** 단어 기반으로 벡터 이미지를 생성하는 데 가장 적합한 라이브러리 또는 API (예: AI 모델 연동 방식)를 심층 조사합니다.
2.  **경쟁사 벤치마킹:** 국내외 유사 픽토그램/아이콘 생성 서비스의 강점과 약점, 가격 정책을 수집합니다.
3.  **픽토그램 데이터셋 구조 조사:** 공공 데이터나 기존 디자인 시스템에서 활용하는 픽토그램의 메타데이터(Metadata) 표준 구조를 조사합니다.

**📱 Youtube / Instagram (마케팅)**
1.  **콘텐츠 컨셉 정의:** 제품 자체 홍보보다는, '건축/디자인 기획력 향상'이라는 상위 목표에 맞춰 콘텐츠 주제를 설정합니다. (예: "PPT로 설득력 있게 만드는 픽토그램 사용법")
2.  **바이럴 콘텐츠 아이디어 3가지:** 타겟 청중(학생, 기획자)이 흥미를 느낄 만한 짧고 시각적인 챌린지 또는 템플릿 활용 아이디어를 구상합니다.
3.  **채널 운영 로드맵 (1개월):** MVP 런칭 전까지의 콘텐츠 업로드 주기와 주제를 간략한 로드맵으로 정리합니다.

#### ✅ 3. 오늘의 최우선 단일 작업 (The Single Most Valuable Task)
현재 가장 부족한 것은 **'구체적인 실행 기준'**입니다. 따라서 모든 에이전트가 참조할 수 있는 **'통합된 디자인 및 기술 스펙'**을 확정하는 것이 오늘 가장 중요합니다.

*   **🔥 작업 목표:** PICTO LAB의 핵심 스타일 시스템과 이를 구현할 기술 스펙을 확정하여, 모든 에이전트의 작업을 구체적인 가이드라인으로 통일합니다.
*   **✨ 기대 결과물:**
    1.  **`Style_System_Spec.pdf`:** (Designer 주도) 모든 픽토그램의 시각적 속성(선의 굵기, 코너 처리, 면적 비율 등)이 정의된 최종 스펙 문서.
    2.  **`Data_Schema_Spec.md`:** (Developer/Researcher 협업) 단어, 스타일 속성, SVG 구조를 포함하는 데이터베이스 스키마 초안.
*   **🧑‍💻 담당 에이전트:** **Designer (리드) & Developer/Researcher (협력)**

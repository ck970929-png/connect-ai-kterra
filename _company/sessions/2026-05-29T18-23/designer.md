### 📌 1. **법규 검토 규칙 워크플로우 정의**  
**단계 1: 단어 입력 시**  
- 입력 단어에 따라 자동 매핑된 법규 체크리스트 활성화 (예: "주차장" → 주차법 제22조)  
- **색상 동적 변환**: `#F4A261` (강조 색상)으로 경고 표시 [근거: designer.md]  

**단계 2: 스타일 선택 시**  
- 선형/면형/굵은 선형 스타일 선택 시, 해당 스타일의 법적 적합성 검토 (예: "면형" → 건축법 제47조 안전 기준)  
- **툴팁 표시**: "색상은 사용자 선호 데이터 기반으로 동적으로 변환됩니다" [근거: designer.md]  

**단계 3: SVG/PNG 다운로드 전**  
- 법적 검토 결과 요약 팝업 알림 (동의 시 다운로드 허용)  
- **레이아웃**: 좌측 스타일 선택 영역, 우측 결과 후보 영역 배치 [근거: designer.md]

### 📌 2. **법적 안전성 알림 UI 디자인 규칙**  
**1. 팝업 알림 스타일**  
- **배경색**: `#2A363B` (브랜드 컬러)  
- **텍스트 색상**: `#F4A261` (동적 강조 색상)  
- **버튼 색상**: `#F4A261` (동의 버튼) + `#2A363B` (취소 버튼)  
- **폰트**: 시스템 기본 폰트 + 14px 크기  

**2. 알림 콘텐츠 구조**  
```markdown
[법규 위반 가능성 경고]  
"선택하신 스타일(면형)은 주차법 제22조에 따라 장애인용 주차 공간 의무 사항을 위반할 수 있습니다.  
[동의] [취소]"
```  
- **툴팁 추가**: "색상은 사용자 선호 데이터 기반으로 동적으로 변환됩니다" [근거: designer.md]

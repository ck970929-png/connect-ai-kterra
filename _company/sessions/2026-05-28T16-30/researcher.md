### 📌 **PICTO LAB 추천 시스템 개선안 (데이터 기반 설계)**  
1. **선호 패턴 분석**  
   - **데이터 소스**: 사용자 단어 입력 기록, SVG 다운로드 이력, 스타일 선택 빈도  
   - **분석 방법**:  
     - **유사 도안 그룹화**: KNN 알고리즘으로 시각적 유사도 기반 추천 (예: "도서관" → "도서관 라이브러리", "도서관 기호")  
     - **사용 빈도 순위화**: 다운로드 빈도 + 스타일 선택 빈도 가중치로 순위 매기기 (예: "면형" 스타일이 "도서관"에 80% 선택 시 우선 노출)  

2. **법규 체크리스트 연동**  
   - **GDPR/CCPA 준수**:  
     - 사용자 동의 필수 (단어 입력 시 "데이터 수집 동의" 팝업 필수)  
     - 데이터 익명화 처리 (사용자 ID 대신 해시값 저장)  
   - **데이터 저장소**:  
     - `./src/data/user-preference.json`에 저장 (예: `{ "도서관": { "스타일": "면형", "다운로드_빈도": 15 } }`)  

3. **추천 규칙 구조화**  
   - **파일 위치**: `./lib/recommendation-rules.ts`  
   - **코드 예시**:  
     ```typescript
     const recommendationRules = {
       "도서관": {
         "스타일": ["면형", "선형"],
         "추천_조건": (userHistory: any) => userHistory.면형_빈도 > 5,
       },
       "벤치": {
         "스타일": ["굵은 선형"],
         "추천_조건": (userHistory: any) => userHistory.선형_빈도 > 10,
       },
     };
     ```  

4. **테스트 시나리오**  
   - **테스트 데이터**: 20개 단어 데이터셋 (예: "도서관", "정원", "도서관", "정원", "도서관")  
   - **결과 예시**:  
     - "도서관" → "면형" 스타일 4개 후보 추천  
     - "정원" → "선형" 스타일 3개 후보 추천

### 🔍 1. **사용자 경험(UX) 개선 제안**  
#### ✅ **문제점 1: 온보딩 부족**  
- **현황**: 단어 입력 → 결과물 생성 흐름만 제공. 사용자가 '왜 이 도구가 필요하나?'를 인지하지 못함.  
- **해결 방안**:  
  - **3단계 인터랙티브 튜토리얼** 추가:  
    1. **문제 제기**: "건축 도면에 맞는 픽토그램이 없어? (예: '경고' 키워드로 '⚠️' 아이콘 생성)"  
    2. **문맥 분석**: "AI가 단어를 분석해 스타일(선형/면형)과 색상 추천"  
    3. **결과 확인**: "SVG/PNG 다운로드 시 도면 배경 모드 자동 적용"  
  - **UI 변경**: 튜토리얼 레이어에 **'#F4A261' 강조 색상**과 **'도면 배경 예시' 이미지** 시각화  

#### ✅ **문제점 2: 실패 시 피드백 부족**  
- **현황**: '법규 위반 키워드'나 '추천 실패' 시 단순 오류 메시지 제공.  
- **해결 방안**:  
  - **AI 실패 로그 컴포넌트** 구현:  
    - **예시**:  
      ```  
      ⚠️ "['법규 위반 키워드: '도배' detected. 대체 키워드: '공간 활용', '정치적 메시지' 추천"  
      ```  
    - **UI 변경**: 오류 메시지에 **'대체 키워드 제안' 팝업** + **'색상/스타일 매핑표'** 표시  
  - **색상 적용**: 오류 메시지 배경색 `#2A363B` (검증된 지식 참조)  

#### ✅ **문제점 3: 결과물 일관성 미흡**  
- **현황**: 생성된 픽토그램이 도면과의 간격, 색상 팔레트에서 이질감 발생.  
- **해결 방안**:  
  - **'도면 배경 모드' 추가**:  
    - **기능**: SVG/PNG 미리보기 시 **'도면 배경 색상(예: #2A363B)'** 자동 적용  
    - **레이아웃**:  
      ```  
      [도면 배경] | [픽토그램]  
      ----------------------------  
      (여백 10px) | (선두께 3px, 색상 #F4A261)  
      ```  
  - **도면 레이아웃**: '용도-스타일 매핑' 표시 (예: '경고' → 면형, '성공' → 굵은 선형)

### 🔍 **1. 사용자 경험(UX) 개선 필수 항목**  
#### ✅ **문제 1: 온보딩 부족**  
- **현황**:  
  - 사용자가 '단어 입력 → 결과물 생성' 흐름만 제공받아, 서비스 가치 인지 실패 가능성.  
  - 경쟁사 대비 차별점 미제시.  
- **수정 지시**:  
  - **3단계 인터랙티브 튜토리얼 추가**:  
    1. **문제 제기**: "건축 도면에 맞는 픽토그램이 없어? (예: '경고' 키워드로 '⚠️' 아이콘 생성)"  
    2. **문맥 분석**: "AI가 단어를 분석해 스타일(선형/면형)과 색상 추천"  
    3. **결과 확인**: "SVG/PNG 다운로드 시 도면 배경 모드 자동 적용"  
  - **UI 변경**: 튜토리얼 레이어에 **`#F4A261` 강조 색상** 및 **도면 배경 예시 이미지** 시각화.  

#### ✅ **문제 2: 실패 시 피드백 부족**  
- **현황**:  
  - '법규 위반 키워드'나 '추천 실패' 시 단순 오류 메시지 제공.  
- **수정 지시**:  
  - **AI 실패 로그 컴포넌트 설계**:  
    - 예시:  
      ```  
      ⚠️ "['법규 위반 키워드: '도배' detected. 대체 키워드: '공간 활용', '정치적 메시지' 추천"  
      ```  
    - 오류 메시지에 **'대체 키워드 제안' 팝업** + **'색상/스타일 매핑표'** 표시.  
  - **색상 적용**: 오류 메시지 배경색 `#2A363B` (검증된 지식 참조).  

#### ✅ **문제 3: 결과물 일관성 미흡**  
- **현황**:  
  - 생성된 픽토그램이 도면과의 간격, 색상 팔레트에서 이질감 발생.  
- **수정 지시**:  
  - **'도면 배경 모드' 추가**:  
    - 기능: SVG/PNG 미리보기 시 **'도면 배경 색상(예: `#2A363B`)'** 자동 적용.  
    - 레이아웃:  
      ```  
      [도면 배경] | [픽토그램]  
      ----------------------------  
      (여백 10px) | (선두께 3px, 색상 `#F4A261`)  
      ```  
  - **도면 레이아웃**: '용도-스타일 매핑' 표시 (예: '경고' → 면형, '성공' → 굵은 선형).

### 🔍 **2. 기술적 검토 및 개선 방향**  
#### ✅ **문제 1: SVG 렌더링 일관성 문제**  
- **현황**:  
  - `svg-renderer.ts`에서 동적 스타일 적용 시, 브라우저 호환성 문제 발생 가능성.  
  - 면형/선형 스타일 경계 처리 부정확 (예: 선두께 3px일 때, 면형 아이콘 외곽선 흐려 보일 수 있음).  
- **수정 방안**:  
  - **SVG 스케일링 최적화**: `viewBox` 동적으로 조정하여 해상도 일관성 유지.  
  - **경계 처리 개선**: `stroke-linejoin` 속성 추가 (예: `round` 또는 `miter`).  
  - **테스트**: [실시간 데이터] 섹션에 `web_preview` 도구로 크로스 브라우저 테스트 실행.  

#### ✅ **문제 2: 스타일 매니저 동적 업데이트 실패**  
- **현황**:  
  - `style-manager.ts`에서 사용자 선택 스타일이 실시간 반영되지 않음 (예: "면형" 선택 후 색상 변경 시 이전 스타일 잔존).  
  - `get_style_sheet` 엔드포인트에서 스타일 데이터 실시간 갱신 실패.  
- **수정 방안**:  
  - **리액티브 상태 관리**: React의 `useState` 또는 `useReducer` 사용해 스타일 변경 시 컴포넌트 재렌더링 강제.  
  - **백엔드 최적화**: `get_style_sheet` 엔드포인트에 캐싱 전략 추가 (예: 5분 간격 갱신).  
  - **도구 실행**: `<run_command>cd ... && py -3 web_preview.py</run_command>`로 실시간 UI 변경 검증.  

#### ✅ **문제 3: 데이터 처리에서의 범위 오버플로우**  
- **현황**:  
  - 사용자 입력 키워드 20자 이상 시 AI 처리 시간 증가 또는 오류 발생 가능성.  
  - 다운로드 시 SVG 파일 크기 커져 브라우저 다운로드 실패 가능성.  
- **수정 방안**:  
  - **입력 제한**: `InputPanel.tsx`에서 키워드 길이 20자 이상일 경우 경고 메시지 표시.  
  - **SVG 최적화**: `download.ts`에서 `svgo` 라이브러리 사용해 압축.  
  - **테스트**: `lint_test.py`로 코드 성능 분석 후, `pack_apply.py`로 패키징 시 압축 기능 검증.

### ⚠️ **예상치 못한 에러 시나리오 및 대응**  
#### ✅ **시나리오 1: AI 모델 키워드 오해**  
- **현황**:  
  - "도면 설계" 입력 시 AI가 "도면" 키워드로 인식해 "도면 배경 모드" 자동 적용, 그러나 스타일 추천 부적절.  
- **수정 방안**:

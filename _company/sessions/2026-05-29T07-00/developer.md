### 🔍 **1. 기능적 버그 및 기술적 한계**  
#### ✅ **문제 1: SVG 렌더링 일관성 문제**  
- **현황**:  
  - `svg-renderer.ts`에서 동적 스타일 적용 시, 브라우저 호환성 문제 발생 가능성 (예: Firefox vs Chrome의 벡터 그림 해상도 차이).  
  - 면형/선형 스타일의 경계 처리가 부정확할 수 있음 (예: 선두께가 3px일 때, 면형 아이콘의 외곽선이 흐려 보일 수 있음).  
- **개선 방안**:  
  - **SVG 스케일링 최적화**: `viewBox` 속성을 동적으로 조정하여 모든 브라우저에서 일관된 해상도 유지.  
  - **경계 처리 개선**: `stroke-linejoin` 속성 추가로 선의 연결부드러움 보장 (예: `round` 또는 `miter`).  
  - **테스트**: [실시간 데이터] 섹션에 `web_preview` 도구로 크로스 브라우저 테스트 실행.  

#### ✅ **문제 2: 스타일 매니저 동적 업데이트 실패**  
- **현황**:  
  - `style-manager.ts`에서 사용자 선택 스타일이 실시간으로 반영되지 않을 수 있음 (예: "면형" 선택 후 색상 변경 시, 이전 스타일이 잔존).  
  - `get_style_sheet` 엔드포인트에서 스타일 데이터가 실시간으로 갱신되지 않음.  
- **개선 방안**:  
  - **리액티브 상태 관리**: React의 `useState` 또는 `useReducer`를 사용해 스타일 변경 시 컴포넌트 재렌더링 강제.  
  - **백엔드 최적화**: `get_style_sheet` 엔드포인트에서 캐싱 전략 추가 (예: 5분 간격 갱신).  
  - **도구 실행**: `<run_command>cd ... && py -3 web_preview.py</run_command>`로 실시간 UI 변경 검증.  

#### ✅ **문제 3: 데이터 처리에서의 범위 오버플로우 (Overflow)**  
- **현황**:  
  - 사용자가 과도한 키워드(예: 100자 이상)를 입력 시, AI 모델이 처리 시간 증가 또는 오류 발생 가능.  
  - 다운로드 시 SVG 파일 크기가 너무 커져서 브라우저에서 다운로드 실패 가능성.  
- **개선 방안**:  
  - **입력 제한**: `InputPanel.tsx`에서 키워드 길이 20자 이상일 경우 경고 메시지 표시.  
  - **SVG 최적화**: `download.ts`에서 SVG 압축 기능 추가 (예: `svgo` 라이브러리 사용).  
  - **테스트**: `lint_test.py`로 코드 성능 분석 후, `pack_apply.py`로 패키징 시 압축 기능 검증.

### 🔍 **2. 예상치 못한 에러 시나리오**  
#### ✅ **시나리오 1: AI 모델이 키워드를 이해하지 못할 때**  
- **현황**:  
  - 사용자가 "도면 설계"와 같은 모호한 키워드 입력 시, AI가 "경고" 또는 "성공"으로 잘못 분류할 수 있음.  
  - 예: "도면 설계" → AI가 "도면"을 키워드로 인식해 "도면 배경 모드"로 자동 적용하지만, 스타일 추천이 부적절할 수 있음.  
- **개선 방안**:  
  - **추천 키워드 시스템**: `get_style_sheet` 엔드포인트에서 키워드 유사도 검색 기능 추가 (예: TF-IDF 또는 BERT 모델).  
  - **사용자 피드백**: 결과물에 "이 키워드로 생성된 아이콘" 대신 "추천 키워드: A, B, C" 표시.  

#### ✅ **시나리오 2: 다운로드 시 PNG 파일 생성 실패**  
- **현황**:  
  - SVG 파일은 생성 가능하나, PNG로 변환 시 텍스처 손실 또는 색상 흐림 발생 가능.  
- **개선 방안**:  
  - **PNG 변환 최적화**: `download.ts`에서 `canvas` API를 사용해 SVG를 PNG로 변환 시, `imageSmoothingEnabled` 설정 추가.  
  - **도구 실행**: `web_preview.py`로 PNG 변환 결과 확인.

### 🔍 **3. 사용자 불편 요소 및 개선 방향**  
#### ✅ **문제 1: 결과물의 색상 팔레트 일관성 부족**  
- **현황**:  
  - 사용자가 "경고" 키워드로 생성한 아이콘은 `#F4A261` 색상이지만, 도면 배경 색상 `#2A363B`과의 조합이 자연스럽지 않을 수 있음.  
- **개선 방안**:  
  - **자동 색상 조합**: `style-manager.ts`에서 키워드에 따라 색상 팔레트 자동 생성 (예: "경고" → `#F4A261` + `#2A363B` 대비).  
  - **도구 실행**: `web_preview.py`로 색상 조합 시각화 테스트.  

#### ✅ **문제 2: 실패 시 피드백 부족**  
- **현황**:  
  - AI가 "법규 위반 키워드"를 감지했을 때, 단순히 "오류" 메시지만 제공해 사용자가 대체 키워드를 모른 채 이탈할 수 있음.  
- **개선 방안**:  
  - **대체 키워드 제안**: `get_style_sheet` 엔드포인트에서 키워드 검증 시, 대체 키워드 리스트 자동 제공 (예: "도배" → "공간 활용", "정치적 메시지").  
  - **UI 변경**: 오류 메시지에 **"대체 키워드 제안" 팝업** 추가.

#### 📌 1. 사용자 선택 패턴  
1. **용도별 스타일 선호도**  
   - **건축/설계 용도:** "벽", "지붕" 등 단어 → **면형 + 곡률 10% + 흰색 배경** [근거: 2026-05-29T06-07/researcher.md]  
   - **산업/기계 용도:** "펌프", "로터리" 등 단어 → **선형 + 파랑 배경 + 곡률 15%** [근거: 2026-05-29T06-22/researcher.md]  
   - **법규 제한:** "벽" 단어는 **선형 스타일 제한** (건축법 기준) [근거: 2026-05-29T05-22/researcher.md]  

2. **시간대별 추천 우선순위**  
   - **금요일~토요일:** **면형 스타일 우선** (시각적 안정성 강조)  
   - **일요일:** **선형 스타일 우선** (다이내믹한 디자인 강조) [근거: 2026-05-29T00-25/researcher.md]  

3. **사용자 빈도 기반 추천**  
   - **고빈도 사용자:** **면형 + 곡률 10%** 우선 (패턴 일관성 강화)  
   - **저빈도 사용자:** **선형 + 파랑 배경** 추천 (신선함 제공) [근거: 2026-05-29T06-22/researcher.md]

#### 📌 2. 추천 알고리즘 평가  
1. **성공 요소**  
   - **단어-스타일 매핑 규칙:** "도서관 분류법"식 그룹화 (유사 도안 집중) [근거: 2026-05-28T16-00/researcher.md]  
   - **법규 체크리스트 연동:** 특정 단어에 대한 스타일 제한 (예: "벽" → 선형 제외) [근거: 2026-05-29T05-22/researcher.md]  
   - **시각적 유사도 점수:** 사용자 색감과 단어의 시각적 연관성 기반 추천 [근거: 2026-05-29T06-07/researcher.md]  

2. **개선 방향**  
   - **시간대 기반 동적 조정:** 금요일~일요일 스타일 우선순위를 알고리즘에 명시화 (예: `if weekday in [5,6] => face_style = 3D`)  
   - **사용자 세분화:** 빈도/신규 사용자 기준으로 스타일 권장 사항 분리 (예: `if user_frequency > 5 => recommend_face_style = 3D`)  
   - **데이터 기반 확장:** 3개월 사용 로그 분석으로 **단어-스타일 빈도 랭킹** 추가 (예: "도어" → 3D, "펌프" → 선형) [근거: 2026-05-28T19-32/researcher.md]

#### 📌 3. 우리 서비스의 강점과 보완 방향  
**강점:**  
- **일관성 강화:** 스타일 규칙과 법규 체크리스트로 디자인 통일성 확보 (vs. Pictographic.ai의 일관성 문제) [근거: 2026-05-29T05-22/researcher.md]  
- **전문성 강조:** 산업별(의료, 금융) 전용 아이콘 라이브러리 제공 (vs. Pictographic.ai의 대중성 중심) [근거: 2026-05-29T00-25/researcher.md]  
- **AI 컨텍스트 이해:** 사용자의 요구사항을 문장으로 분석해 맞춤형 추천 (vs. Pictographic.ai의 단순 검색 필터링) [근거: 2026-05-28T16-00/researcher.md]  

**보완 방향:**  
- **커스터마이징 확장:** 사용자가 3D/선형 스타일 변형을 요청 시, AI가 자동으로 디자인 파라미터 조정 (예: `if user_request = "3D" => adjust_curve = 10%`)  
- **실시간 데이터 수집:** 사용자 선택 패턴을 기반으로 **단어-스타일 빈도 랭킹** 실시간 업데이트 (예: 3개월 데이터 분석 후 매주 랜딩 페이지에 반영) [근거: 2026-05-28T19-52/researcher.md]  
- **법규 체크리스트 확장:** 지역별 건축법/주차장법 기준으로 스타일 제한 규칙 자동 적용 (예: "벽" 단어 → 한국 건축법 기준 선형 제외)

### 4. 시스템 아키텍처 및 API 설계  
**기술 스택**: FastAPI (백엔드), React (프론트엔드), PostgreSQL (DB), Redis (캐싱)  

**핵심 컴포넌트**:  
1. **실패 로그 수집 API**  
   - **엔드포인트**: `POST /api/v1/failure-logs`  
   - **요청 형식**:  
     ```json
     {
       "keyword": "지하주차장",
       "error_type": "법규 위반",
       "timestamp": "2026-05-29T14:30:00Z"
     }
     ```  
   - **기능**: 로그 저장 (PostgreSQL), Redis 캐싱  

2. **주간 분석 프로세스**  
   - **스케줄링**: 매주 금요일 자동 실행 (Cron Job)  
   - **분석 로직**:  
     - 도서관 분류법 (예: '도서관' → 면형 75%)  
     - 법규 체크리스트 연동 (법제처 API)  

3. **규칙 업데이트 API**  
   - **엔드포인트**: `PUT /api/v1/rules`  
   - **요청 형식**:  
     ```json
     {
       "version": "1.2.0",
       "rule_updates": [
         {"keyword": "지하주차장", "action": "필터링", "replacement": "주차장"},
         {"keyword": "문", "style_priority": {"선형": 0.7, "면형": 0.3}}
       ]
     }
     ```  
   - **기능**: 규칙 DB 업데이트 (PostgreSQL), Redis 캐시 무효화  

4. **프론트엔드 동기화**  
   - **엔드포인트**: `GET /api/v1/rules`  
   - **응답 형식**:  
     ```json
     {
       "current_version": "1.2.0",
       "rules": [
         {"keyword": "지하주차장", "status": "필터링", "replacement": "주차장"},
         {"keyword": "문", "style_priority": {"선형": 0.7, "면형": 0.3}}
       ]
     }
     ```  
   - **기능**: React에서 실시간 규칙 로드 (SWR 또는 useEffect)

### 🧠 **1. 시스템 아키텍처 설계**  
**목표**: 실패 로그 자동 수집 → 분석 → 규칙 업데이트 → 버전 관리 → 프론트엔드 동기화  
**기술 스택**: FastAPI (백엔드), React (프론트엔드), PostgreSQL (DB), Redis (캐싱)  

---

### 🛠️ **2. 핵심 컴포넌트 구현**  

#### **2.1 실패 로그 수집 API**  
- **엔드포인트**: `POST /api/v1/failure-logs`  
- **요청 형식**:  
  ```json
  {
    "keyword": "지하주차장",
    "error_type": "법규 위반",
    "timestamp": "2026-05-29T14:30:00Z"
  }
  ```  
- **기능**:  
  - 로그 저장 (PostgreSQL)  
  - Redis에 실시간으로 캐시 (분석 속도 향상)  
  - **근거**: `researcher.md` (2026-05-29T05-22)  

---

#### **2.2 주간 분석 프로세스**  
- **스케줄링**: 매주 금요일 자동 실행 (Cron Job)  
- **분석 로직**:  
  1. **도서관 분류법**: 키워드 유사도 계산 (예: "도서관" → 면형 스타일 75%)  
  2. **법규 체크리스트**: 법제처 API 연동 (예: "지하주차장" → 건축법 제12조 자동 적용)  
  3. **데이터 기반 규칙 생성**: 사용자 선호 데이터 기반 색상/스타일 매핑  
- **출력**: `analysis_report.json` (규칙 변경 사항)  
- **근거**: `designer.md` (2026-05-29T04-12)  

---

#### **2.3 규칙 업데이트 API**  
- **엔드포인트**: `PUT /api/v1/rules`  
- **요청 형식**:  
  ```json
  {
    "version": "1.2.0",
    "rule_updates": [
      {"keyword": "지하주차장", "action": "필터링", "replacement": "주차장"},
      {"keyword": "문", "style_priority": {"선형": 0.7, "면형": 0.3}}
    ]
  }
  ```  
- **기능**:  
  - 규칙 DB 업데이트 (PostgreSQL)  
  - Redis 캐시 무효화 (실시간 반영)  
  - **버전 관리**: SemVer (예: `1.2.0`)  
- **근거**: `developer.md` (2026-05-28T13-50)  

---

#### **2.4 프론트엔드 동기화**  
- **엔드포인트**: `GET /api/v1/rules`  
- **응답 형식**:  
  ```json
  {
    "current_version": "1.2.0",
    "rules": [
      {"keyword": "지하주차장", "status": "필터링", "replacement": "주차장"},
      {"keyword": "문", "style_priority": {"선형": 0.7, "면형": 0.3"}}
    ]
  }
  ```  
- **기능**:  
  - React에서 실시간 규칙 로드 (SWR 또는 useEffect)  
  - 사용자 피드백 반영 (예: "스마트폰 충전소" → 선형 스타일)  
- **근거**: `researcher.md` (2026-05-29T00-10)

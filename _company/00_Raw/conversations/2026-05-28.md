## 🚀 1. 프로젝트 개요
K-TERRA ARCHI LAB은 AI 기반의 건축·디자인 작업실이자, 사용자가 단어만 입력하면 통일된 픽토그램을 생성하는 서비스를 만드는 1인 크리에이터 회사입니다.

**핵심 목표:** AI 기반의 전문적인 디자인 결과물(픽토그램, 건축 도면 등)을 생성하고, 이를 통해 사용자에게 가치를 제공하는 플랫폼 구축.

## 🎨 2. 핵심 서비스: PICTO LAB
*   **기능:** 사용자가 입력한 일반 단어(예: '계단', '문', '나무')를 AI가 해석하여, 통일된 스타일의 픽토그램으로 변환 및 생성합니다.
*   **가치:** 기존의 상용 픽토그램 라이브러리에서 부족했던 '통일성'과 '맥락적 적합성'을 AI가 채워줍니다.
*   **기술적 요구사항:** 단어 입력 → AI 해석 → 픽토그램 생성 (SVG/PNG 출력)

## 📐 3. 디자인 시스템 및 통일성
*   **필수 원칙:** 모든 픽토그램은 '통일된 스타일 시스템'을 적용받아야 합니다. (예: 선의 굵기, 모서리 처리, 투사 방식 등)
*   **스타일 옵션:**
    *   선형 (Line Art): 가장 기본적인 형태.
    *   면형 (Filled): 채워진 형태.
    *   굵은 선형 (Bold Line): 강조를 위한 형태.
*   **출력 포맷:**
    *   **SVG:** 벡터 기반의 원본 파일 (편집 가능, 필수).
    *   **PNG:** 범용적인 이미지 파일 (다운로드용).

## 🧑‍💻 4. 타겟 청중 및 시장

### 📄 PICTO LAB MVP 구축을 위한 핵심 가이드라인 초안

사장님, 아래와 같이 3가지 핵심 영역을 중심으로 가이드라인을 초안화했습니다. 이 문서는 모든 에이전트의 작업 기준점이 됩니다.

#### 💡 1. 프로젝트 목표 및 사용자 경험 (UX/Business Focus)
*   **핵심 서비스:** 사용자가 단어(Text Input)만 입력하여 AI 기반의 전문 픽토그램을 생성/다운로드하는 솔루션.
*   **핵심 가치:** 건축/디자인 분야의 통일된 시각 언어(Visual Language) 제공.
*   **필수 플로우:**
    1.  **단어 입력:** 사용자가 키워드 입력.
    2.  **AI 추천/검색:** AI가 키워드에 맞는 픽토그램을 추천하거나 검색.
    3.  **스타일 적용:** 선형/면형/굵은 선형 등 정의된 스타일 시스템을 자동으로 적용하여 픽토그램 생성.
    4.  **다운로드:** SVG (벡터) 및 PNG (범용) 형태로 다운로드.

#### 🎨 2. 디자인 및 스타일 시스템 (Design Focus)
*   **스타일 원칙:** 모든 픽토그램은 통일된 시각 가이드라인을 준수해야 합니다.
    *   **선 굵기 (Stroke Weight):** 일관된 굵기 (예: 2pt) 사용.
    *   **모서리 처리:** 부드러운 곡률 (Radius) 적용.
    *   **구조:** 불필요한 디테일 제거, 최소한의 정보 전달에 집중 (Minimalism).
*   **스타일 시스템화:** 선형/면형/굵은 선형 등 여러 스타일 옵션을 제공하되, 각 스타일의 규칙을 명확히 정의하고 관리하는 **Style Sheet**가 필수적입니다.

#### 💻 3. 기술 및 데이터 구조 (Developer Focus)
*   **기술 스택:** 웹 기반 (React/Next.js 추천), SVG 생성 엔진 필수.
*   **데이터 구조 (Source of Truth):**
    *   `{word: string, meaning: string, style: string, svg_data: string, recommended_icons: []}`
    *   단순히 이미지를 저장하는 것이 아니라, **'단어-의미-SVG 속성'**을 포함하는 표준화된 데이터 구조가 필요합니다.
*   **최소 기능 구현 (MVP Scope):**
    *   단어 입력 및 픽토그램 검색/생성 (기본 기능).
    *   SVG/PNG 다운로드 (핵심 출력).
    *   *(선택적 기능: 사용자별 저장/로그인 등은 2단계 목표로 이월)*

### 📄 PICTO LAB MVP 구축을 위한 핵심 가이드라인 초안

사장님, 아래와 같이 3가지 핵심 영역을 중심으로 가이드라인을 초안화했습니다. 이 문서는 모든 에이전트의 작업 기준점이 됩니다.

#### 💡 1. 프로젝트 목표 및 사용자 경험 (UX/Business Focus)
*   **핵심 서비스:** 사용자가 단어(Text Input)만 입력하여 AI 기반의 전문 픽토그램을 생성/다운로드하는 솔루션.
*   **핵심 가치:** 건축/디자인 분야의 통일된 시각 언어(Visual Language) 제공.
*   **필수 플로우:**
    1.  **단어 입력:** 사용자가 키워드 입력.
    2.  **AI 추천/검색:** AI가 키워드에 맞는 픽토그램을 추천하거나 검색.
    3.  **스타일 적용:** 선형/면형/굵은 선형 등 정의된 스타일 시스템을 자동으로 적용하여 픽토그램 생성.
    4.  **다운로드:** SVG (벡터) 및 PNG (범용) 형태로 다운로드.

#### 🎨 2. 디자인 및 스타일 시스템 (Design Focus)
*   **스타일 원칙:** 모든 픽토그램은 통일된 시각 가이드라인을 준수해야 합니다.
    *   **선 굵기 (Stroke Weight):** 일관된 굵기 (예: 2pt) 사용.
    *   **모서리 처리:** 부드러운 곡률 (Radius) 적용.
    *   **구조:** 불필요한 디테일 제거, 최소한의 정보 전달에 집중 (Minimalism).
*   **스타일 시스템화:** 선형/면형/굵은 선형 등 여러 스타일 옵션을 제공하되, 각 스타일의 규칙을 명확히 정의하고 관리하는 **Style Sheet**가 필수적입니다.

#### 💻 3. 기술 및 데이터 구조 (Developer Focus)
*   **기술 스택:** 웹 기반 (React/Next.js 추천), SVG 생성 엔진 필수.
*   **데이터 구조 (Source of Truth):**
    *   `{word: string, meaning: string, style: string, svg_data: string, recommended_icons: []}`
    *   단순히 이미지를 저장하는 것이 아니라, **'단어-의미-SVG 속성'**을 포함하는 표준화된 데이터 구조가 필요합니다.
*   **최소 기능 구현 (MVP Scope):**

### 💡 Task 1: PICTO LAB 기술 명세서 최종 확정 (Tech Spec)
*   **목표:** 단어 입력 → AI 검색/추천 → SVG/PNG 출력까지의 모든 과정에 대한 기술적 흐름과 데이터 구조를 확정합니다.
*   **핵심 내용:**
    *   **데이터 구조:** '단어-의미-SVG 속성'의 표준화된 구조를 확정하고, 데이터베이스 스키마 초안을 작성합니다.
    *   **API 플로우:** 프론트엔드(입력)와 백엔드(AI 처리/SVG 생성) 간의 통신 프로토콜을 정의합니다.
*   **담당 에이전트:**
    *   **Developer:** 기술 아키텍처 담당 (Backend/API 플로우 설계)
    *   **Researcher:** 기술 난이도 분석 및 구현 가능성 검증
    *   **Designer:** 출력물(SVG)의 기술적 제약 사항 검토 및 반영

### 📐 Task 2: 픽토그램 스타일 시스템 가이드라인 확정 (Style Sheet)
*   **목표:** 통일성을 보장하는 시각적 규칙(선의 굵기, 곡률, 모서리 처리 등)을 측정 가능한 가이드라인으로 최종 확정합니다.
*   **핵심 내용:**
    *   **규칙 정의:** 모든 픽토그램이 따를 '선 가중치(Line Weight)', '곡률 반경(Curvature)', '모서리 처리 방식(Corner Radius)'을 수치적으로 확정합니다.
    *   **예외 처리:** 스타일 시스템을 위반할 수 있는 예외적인 상황(예: 복합 요소)에 대한 처리 규칙을 정의합니다.
*   **담당 에이전트:**
    *   **Designer:** 가이드라인의 시각적 완결성 검토 및 최종 초안 작성
    *   **Researcher:** 디자인 원칙에 대한 학술적/전문적 근거 제시
    *   **Writer:** Style Sheet 문서화 및 명료한 규칙서 작성

### 📈 Task 3: MVP 시장 검증 핵심 KPI 재확인 (Market Validation)
*   **목표:** MVP 출시 후, 서비스의 성공 여부를 측정할 핵심 지표(KPI)를 재확인하고, 검증을 위한 가설을 세웁니다.
*   **핵심 내용:**
    *   **핵심 지표:** '재방문율(Retention)', '특정 단어 검색 성공률(Search Success Rate)', 'SVG 다운로드 전환율' 등을 최우선 KPI로 확정합니다.
    *   **검증 플로우:** 어떤 사용자 그룹(Persona)에게, 어떤 시점에 KPI를 측정할지 시나리오를 정의합니다.
*   **담당 에이전트:**
    *   **Business:** 비즈니스 모델 관점에서의 수익화 및 지표 연계 검토
    *   **Writer:** KPI 정의의 명확화 및 사용자 관점의 가설 수립
    *   **Researcher:** 시장 데이터 및 경쟁사 분석 기반의 KPI 보강

```python
# backend/main.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import List

app = FastAPI(title="AI Image Generator API")

# API 요청 본문 구조 정의
class QueryRequest(BaseModel):
    text_input: str
    style: str = "minimal"
    style_weights: List[float] = []

# API 응답 본문 구조 정의
class GenerationResponse(BaseModel):
    success: bool
    message: str
    generated_image_url: str
    metadata: dict

@app.post("/api/v1/generate", response_model=GenerationResponse)
async def generate_image(query: QueryRequest):
    """
    사용자가 입력한 텍스트와 스타일을 기반으로 가상의 이미지를 생성합니다.
    """
    print(f"Received request: Text='{query.text_input}', Style='{query.style}'")

    if not query.text_input:
        raise HTTPException(status_code=400, detail="텍스트 입력을 해주세요.")

    # --- 핵심 비즈니스 로직 시뮬레이션 ---
    # 실제로는 여기에 Stable Diffusion, DALL-E 등 외부 API 호출 로직이 들어갑니다.
    try:
        # 가상의 이미지 생성 성공 처리
        mock_url = f"https://mock-ai-service.com/images/{hash(query.text_input + query.style)}.png"
        return GenerationResponse(
            success=True,
            message=f"'{query.text_input}'에 대한 '{query.style}' 스타일 이미지가 성공적으로 생성되었습니다.",
            generated_image_url=mock_url,
            metadata={"input_text": query.text_input, "style": query.style}
        )
    except Exception as e:
        return GenerationResponse(
            success=False,
            message=f"이미지 생성 중 오류가 발생했습니다: {str(e)}",
            generated_image_url="",
            metadata={}
        )

@app.get("/health")
def read_root():
    return {"status": "ok", "service": "AI Image Generator API"}
```

```jsx
// frontend/src/App.jsx
import React, { useState } from 'react';
import './App.css';

function App() {
  const [prompt, setPrompt] = useState('');
  const [style, setStyle] = useState('minimal');
  const [result, setResult] = useState(null);
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (e) => {
    e.preventDefault();
    setLoading(true);
    setResult(null);

    try {
      const response = await fetch('http://localhost:8000/api/v1/generate', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          text_input: prompt,
          style: style,
          style_weights: []
        }),
      });

      const data = await response.json();
      setResult(data);

    } catch (error) {
      console.error("API 호출 오류:", error);
      setResult({ success: false, message: "네트워크 연결을 확인해주세요." });
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="App">
      <header>
        <h1>✨ AI 이미지 생성기</h1>
        <p>텍스트 프롬프트와 원하는 스타일을 입력하세요.</p>
      </header>

      <form onSubmit={handleSubmit} className="input-form">
        <input
          type="text"
          placeholder="생성하고 싶은 이미지에 대한 상세 설명 (예: 우주를 유영하는 고양이)"
          value={prompt}
          onChange={(e) => setPrompt(e.target.value)}
          disabled={loading}
        />
        <select value={style} onChange={(e) => setStyle(e.target.value)} disabled={loading}>
          <option value="minimal">미니멀리즘 (Minimal)</option>
          <option value="cinematic">시네마틱 (Cinematic)</option>
          <option value="watercolor">수채화 (Watercolor)</option>
        </select>
        <button type="submit" disabled={loading || !prompt}>
          {loading ? '생성 중...' : '이미지 생성하기'}
        </button>
      </form>

      {result && (
        <div className={`result-box ${result.success ? 'success' : 'error'}`}>
          <h2>{result.success ? '✅ 성공' : '❌ 실패'}</h2>
          <p>{result.message}</p>
          {result.success && (
            <div>
              <img src={result.generated_image_url} alt="Generated" style={{maxWidth: '100%', maxHeight: '300px', objectFit: 'contain', border: '1px solid #ccc'}} />
              <p>Meta: {JSON.stringify(result.metadata)}</p>
            </div>
          )}
        </div>
      )}
    </div>
  );
}

export default App;
```

```jsx
// frontend/src/App.jsx
import React, { useState } from 'react';
import './App.css';

function App() {
  const [prompt, setPrompt] = useState('');
  const [style, setStyle] = useState('minimal');
  const [result, setResult] = useState(null);
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (e) => {
    e.preventDefault();
    setLoading(true);
    setResult(null);

    try {
      const response = await fetch('http://localhost:8000/api/v1/generate', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          text_input: prompt,
          style: style,
          style_weights: []
        }),
      });

      const data = await response.json();
      setResult(data);

    } catch (error) {
      console.error("API 호출 오류:", error);
      setResult({ success: false, message: "네트워크 연결을 확인해주세요." });
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="App">
      <header>
        <h1>✨ AI 이미지 생성기</h1>
        <p>텍스트 프롬프트와 원하는 스타일을 입력하세요.</p>
      </header>

      <form onSubmit={handleSubmit} className="input-form">
        <input
          type="text"
          placeholder="생성하고 싶은 이미지에 대한 상세 설명 (예: 우주를 유영하는 고양이)"
          value={prompt}
          onChange={(e) => setPrompt(e.target.value)}
          disabled={loading}
        />
        <select value={style} onChange={(e) => setStyle(e.target.value)} disabled={loading}>
          <option value="minimal">미니멀리즘 (Minimal)</option>
          <option value="cinematic">시네마틱 (Cinematic)</option>
          <option value="watercolor">수채화 (Watercolor)</option>
        </select>
        <button type="submit" disabled={loading || !prompt}>
          {loading ? '생성 중...' : '이미지 생성하기'}
        </button>
      </form>

      {result && (
        <div className={`result-box ${result.success ? 'success' : 'error'}`}>
          <h2>{result.success ? '✅ 성공' : '❌ 실패'}</h2>
          <p>{result.message}</p>
          {result.success && (
            <div>
              <img src={result.generated_image_url} alt="Generated" style={{maxWidth: '100%', maxHeight: '300px', objectFit: 'contain', border: '1px solid #ccc'}} />
              <p>Meta: {JSON.stringify(result.metadata)}</p>
            </div>
          )}
        </div>
      )}
    </div>
  );
}

export default App;
```

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import List, Dict

# FastAPI 인스턴스 생성
app = FastAPI(
    title="Asset Management API",
    description="제품 에셋 검색, 상세 조회, 스타일 정보를 제공하는 백엔드 API."
)

# --- 1. 데이터 모델 정의 (Pydantic Schema) ---

# 검색 요청 바디 모델
class SearchRequest(BaseModel):
    keyword: str
    category: str = None

# 에셋 상세 정보 모델
class AssetDetail(BaseModel):
    asset_id: str
    name: str
    description: str
    file_url: str
    usage_rights: str

# 스타일 정보 모델
class StyleInfo(BaseModel):
    style_name: str
    style_code: str
    description: str
    compatibility: List[str]

# --- 2. API 엔드포인트 정의 ---

@app.get("/")
def read_root():
    """루트 경로 테스트용 엔드포인트"""
    return {"message": "Asset Management API is running!"}

# -----------------------------------------------------
# 🎨 Endpoint 1: 에셋 검색 기능 (Search Asset)
# -----------------------------------------------------
@app.post("/api/v1/assets/search", response_model=List[AssetDetail])
def search_assets(request: SearchRequest):
    """
    키워드와 카테고리를 기반으로 에셋을 검색합니다.
    실제 DB 조회 로직을 시뮬레이션합니다.
    """
    print(f"--- 검색 요청 수신: 키워드='{request.keyword}', 카테고리='{request.category}' ---")
    
    # 가상의 검색 결과 데이터 (실제로는 DB 쿼리 필요)
    if "logo" in request.keyword.lower() and request.category == "branding":
        return [
            AssetDetail(
                asset_id="A001",
                name="Company Main Logo (RGB)",
                description="주요 브랜딩용 로고 파일. 모든 매체에 사용 가능.",
                file_url="http://api.example.com/asset/logo_main.png",
                usage_rights="Commercial Use Allowed"
            ),
            AssetDetail(
                asset_id="A002",
                name="Logo Icon (Favicon)",
                description="웹사이트의 파비콘으로 사용되는 작은 아이콘.",
                file_url="http://api.example.com/asset/logo_icon.svg",
                usage_rights="Web Use Only"
            )
        ]
    elif "background" in request.keyword.lower():
        return [
            AssetDetail(
                asset_id="A003",
                name="Abstract Background V2",
                description="2024년 신규 캠페인용 추상적 배경 이미지.",
                file_url="http://api.example.com/asset/bg_v2.jpg",
                usage_rights="Internal Use Only"
            )
        ]
    else:
        # 검색 결과가 없을 경우 빈 리스트 반환
        return []

# -----------------------------------------------------
# 🖼️ Endpoint 2: 에셋 상세 정보 조회 (Get Asset Detail)
# -----------------------------------------------------
@app.get("/api/v1/assets/detail/{asset_id}", response_model=AssetDetail)
def get_asset_detail(asset_id: str):
    """
    asset_id를 기반으로 에셋의 상세 정보를 조회합니다.
    """
    print(f"--- 상세 조회 요청 수신: Asset ID={asset_id} ---")
    
    # 가상의 상세 데이터 조회 로직
    if asset_id == "A001":
        return AssetDetail(
            asset_id="A001",
            name="Company Main Logo (RGB)",
            description="주요 브랜딩용 로고 파일. 모든 매체에 사용 가능하며, 가이드라인 준수 필수.",
            file_url="http://api.example.com/asset/logo_main.png",
            usage_rights="Commercial Use Allowed"
        )
    elif asset_id == "A999":
        # 에셋을 찾을 수 없을 경우 404 에러 발생
        raise HTTPException(status_code=404, detail=f"Asset with ID '{asset_id}' not found.")
    else:
        return AssetDetail(
            asset_id=asset_id,
            name="Unknown Asset",
            description="해당 에셋에 대한 상세 정보가 없습니다.",
            file_url="",
            usage_rights="N/A"
        )

# -----------------------------------------------------
# ✨ Endpoint 3: 스타일 정보 조회 (Get Style Info)
# -----------------------------------------------------
@app.get("/api/v1/styles/info/{style_code}", response_model=StyleInfo)
def get_style_info(style_code: str):
    """
    스타일 코드를 기반으로 사용 가능한 디자인 스타일 정보를 조회합니다.
    """
    print(f"--- 스타일 정보 조회 요청 수신: Style Code={style_code} ---")
    
    # 가상의 스타일 데이터 조회 로직
    if style_code.upper() == "COLOR_V2":
        return StyleInfo(
            style_name="Vivid Color Palette",
            style_code="COLOR_V2",
            description="채도가 높은 현대적인 색상 팔레트. 디지털 환경에 최적화.",
            compatibility=["Web", "Print (Digital)", "UI Kit"]
        )
    elif style_code.upper() == "MINIMAL_B":
        return StyleInfo(
            style_name="Minimalist Black & White",
            style_code="MINIMAL_B",
            description="가장 기본적인 흑백 조합. 클래식하고 시대를 타지 않는 디자인.",
            compatibility=["Print (High-End)", "Photography"]
        )
    else:
        raise HTTPException(status_code=404, detail=f"Style with code '{style_code}' not found.")
```

### 🎨 3. 픽토그램 스타일 시스템 (Style Sheet) 정의

모든 픽토그램은 아래 세 가지 기본 스타일을 기준으로 생성되어야 하며, 사용자가 원하는 스타일에 따라 전환이 가능해야 합니다.

#### A. 기본 원칙 (Global Rules)
1. **색상 팔레트:** `#333333` (Primary Black/Gray) - 텍스트/선 기본색.
2. **모서리 처리:** 모든 요소는 `Border-radius: 3px`을 기본으로 적용한다. (딱딱한 직각 지양)
3. **벡터 표준:** 모든 결과물은 SVG 포맷을 기본으로 하며, 모든 속성은 명시적 좌표와 벡터 곡선으로 정의한다.

#### B. 3가지 표준 스타일 정의 (The 3 Styles)

| 스타일명 | 용도 | 선 굵기 (Stroke Weight) | 채움 (Fill) | 특징 및 사용 예시 |
| :---: | :---: | :---: | :---: | :---: |
| **1. Line Style (선형)** | 기획, 도면, 흐름, 경량화 | `2px` (가늘고 정교함) | `none` (투명) | *가장 표준적인 스타일.* 가벼운 느낌의 다이어그램이나 레이아웃 표현에 적합. |
| **2. Solid Style (면형)** | 건물 전체, 구조, 오브제 | `0px` (외곽선 없음) | `#333333` (단색) | *가장 견고한 스타일.* 단일 오브제나 전체 구조를 강조할 때 사용. |
| **3. Bold Line Style (굵은 선형)** | 핵심 요소, 강조, 가독성 | `3px` (굵고 시원함) | `none` (투명) | *가장 눈에 띄는 스타일.* 사용자가 즉각적으로 인식해야 하는 핵심 구조(예: 문, 계단)에 사용. |

### 💻 2. 핵심 구현: SVG 생성 유틸리티 (The Core Logic)

SVG 생성은 단순한 문자열 조작이 아닙니다. 단어의 의미론적/시각적 특성(예: '연결'은 화살표, '건축'은 기둥)을 파라미터화된 도형으로 변환하는 복잡한 로직입니다. 저는 이를 `lib/svgGenerator.ts`에 정의하겠습니다.

**주의:** SVG 생성은 외부 라이브러리(예: D3.js 또는 SVG Path Generator)를 활용해야 하므로, 여기서는 인터페이스와 핵심 로직만 정의하고 실제 복잡한 렌더링은 주석으로 대체하겠습니다.

**`src/pages/HomePage.tsx`** (메인 페이지 로직)
```tsx
import React, { useState, useEffect } from 'react';
import DesignSystem from '../components/DesignSystem';
import { DesignConcept, GeneratedSvg } from '../types';
import { generateSvgFromConcept } from '../utils/pathGenerator';

const HomePage: React.FC = () => {
    const [searchTerm, setSearchTerm] = useState('');
    const [results, setResults] = useState<DesignConcept[]>([]);
    const [svgResults, setSvgResults] = useState<GeneratedSvg[]>([]);

    useEffect(() => {
        // Mock 데이터 로딩 시뮬레이션
        const mockConcepts: DesignConcept[] = [
            { id: 'c1', name: '미니멀리즘 (Minimalism)', description: '불필요한 요소를 제거하고 본질에 집중하는 디자인 철학.', keywords: ['simple', 'clean', 'essence'] },
            { id: 'c2', name: '유기적 형태 (Organic Form)', description: '자연에서 영감을 받은 부드럽고 흐르는 듯한 형태.', keywords: ['nature', 'fluid', 'bio'] },
            { id: 'c3', name: '네오-브루탈리즘 (Neo-Brutalism)', description: '거칠고 투박하며 구조적인 요소를 강조하는 접근.', keywords: ['raw', 'concrete', 'structure'] },
        ];
        setResults(mockConcepts);
    }, []);

    useEffect(() => {
        // 검색 로직 실행 시, SVG 생성 로직도 함께 호출
        if (results.length > 0) {
            const newSvgResults: GeneratedSvg[] = results.map(concept => 
                generateSvgFromConcept(concept)
            );
            setSvgResults(newSvgResults);
        }
    }, [results]);

    return (
        <div className="container mx-auto p-8">
            <h1 className="text-4xl font-extrabold text-gray-900 mb-6 border-b pb-2">
                디자인 컨셉 라이브러리
            </h1>

            {/* 검색 바 */}
            <div className="mb-8 p-4 bg-gray-50 rounded-lg shadow-inner">
                <input
                    type="text"
                    placeholder="키워드나 컨셉 이름을 검색해보세요..."
                    value={searchTerm}
                    onChange={(e) => setSearchTerm(e.target.value)}
                    className="w-full p-3 border border-gray-300 rounded-lg focus:ring-indigo-500 focus:border-indigo-500"
                />
            </div>

            {/* 결과 그리드 */}
            <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8">
                {results.length > 0 ? (
                    results.map(concept => (
                        <DesignSystem
                            key={concept.id}
                            concept={concept}
                            svgData={svgResults[results.indexOf(concept)] || { svgString: '', width: 0, height: 0 }}
                        />
                    ))
                ) : (
                    <p className="col-span-3 text-center text-gray-500">검색된 컨셉이 없습니다.</p>
                )}
            </div>
        </div>
    );
};

export default HomePage;
```

### 📝 요약 및 설명

1.  **역할 분리:** 컴포넌트 (`DesignSystem`)는 UI 렌더링에만 집중하고, 비즈니스 로직 (`pathGenerator.ts`)은 SVG 생성이라는 복잡한 계산을 담당합니다.
2.  **데이터 흐름:** `HomePage` 컴포넌트가 상태 관리와 데이터 로딩을 담당합니다. 사용자가 검색을 실행하면, `results` 상태가 업데이트되고, 이 변화를 감지하여 `useEffect` 훅이 트리거되며, 각 컨셉별로 `generateSvgFromConcept` 함수를 호출하여 `svgResults` 상태를 업데이트합니다.
3.  **SVG 렌더링:** `dangerouslySetInnerHTML`을 사용하여 생성된 SVG 문자열을 React에 주입했습니다. 이는 SVG가 순수 HTML 문자열로 취급되기 때문에 필요한 기술입니다.
4.  **확장성:** 이 구조는 새로운 컨셉 데이터나 새로운 디자인 규칙이 추가되어도, `DesignConcept` 인터페이스와 `generateSvgFromConcept` 함수만 수정하면 되므로 매우 확장성이 높습니다.

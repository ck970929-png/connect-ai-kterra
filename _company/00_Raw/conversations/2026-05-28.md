## 🚀 1. 프로젝트 개요
K-TERRA ARCHI LAB은 AI 기반의 건축·디자인 작업실이자, 사용자가 단어만 입력하면 통일된 픽토그램을 생성하는 서비스를 만드는 1인 크리에이터 회사입니다.

**핵심 목표:** AI 기반의 전문적인 디자인 결과물(픽토그램, 건축 도면 등)을 생성하고, 이를 통해 사용자에게 가치를 제공하는 플랫폼 구축.

## 🎨 2. 핵심 서비스: PICTO LAB
*   **기능:** 사용자가 입력한 일반 단어(예: '계단', '문', '나무')를 AI가 해석하여, 통일된 스타일의 픽토그램으로 변환 및 생성합니다.
*   **가치:** 기존의 상용 픽토그램 라이브러리에서 부족했던 '통일성'과 '맥락적 적합성'을 AI가 채워줍니다.
*   **기술적 요구사항:** 단어 입력 → AI 해석 → 픽토그램 생성 (SVG/PNG 출력)

## 📐 3. 디자인 시스템 및 통일성
*   **필수 원칙:** 모든 픽토그램은 '통일된 스타일 시스템'을 적용받아야 합니다. (예: 선의 굵기, 모서리 처리, 투사 방식 등)
*   **스타일 옵션:**
    *   선형 (Line Art): 가장 기본적인 형태.
    *   면형 (Filled): 채워진 형태.
    *   굵은 선형 (Bold Line): 강조를 위한 형태.
*   **출력 포맷:**
    *   **SVG:** 벡터 기반의 원본 파일 (편집 가능, 필수).
    *   **PNG:** 범용적인 이미지 파일 (다운로드용).

## 🧑‍💻 4. 타겟 청중 및 시장

### 📄 PICTO LAB MVP 구축을 위한 핵심 가이드라인 초안

사장님, 아래와 같이 3가지 핵심 영역을 중심으로 가이드라인을 초안화했습니다. 이 문서는 모든 에이전트의 작업 기준점이 됩니다.

#### 💡 1. 프로젝트 목표 및 사용자 경험 (UX/Business Focus)
*   **핵심 서비스:** 사용자가 단어(Text Input)만 입력하여 AI 기반의 전문 픽토그램을 생성/다운로드하는 솔루션.
*   **핵심 가치:** 건축/디자인 분야의 통일된 시각 언어(Visual Language) 제공.
*   **필수 플로우:**
    1.  **단어 입력:** 사용자가 키워드 입력.
    2.  **AI 추천/검색:** AI가 키워드에 맞는 픽토그램을 추천하거나 검색.
    3.  **스타일 적용:** 선형/면형/굵은 선형 등 정의된 스타일 시스템을 자동으로 적용하여 픽토그램 생성.
    4.  **다운로드:** SVG (벡터) 및 PNG (범용) 형태로 다운로드.

#### 🎨 2. 디자인 및 스타일 시스템 (Design Focus)
*   **스타일 원칙:** 모든 픽토그램은 통일된 시각 가이드라인을 준수해야 합니다.
    *   **선 굵기 (Stroke Weight):** 일관된 굵기 (예: 2pt) 사용.
    *   **모서리 처리:** 부드러운 곡률 (Radius) 적용.
    *   **구조:** 불필요한 디테일 제거, 최소한의 정보 전달에 집중 (Minimalism).
*   **스타일 시스템화:** 선형/면형/굵은 선형 등 여러 스타일 옵션을 제공하되, 각 스타일의 규칙을 명확히 정의하고 관리하는 **Style Sheet**가 필수적입니다.

#### 💻 3. 기술 및 데이터 구조 (Developer Focus)
*   **기술 스택:** 웹 기반 (React/Next.js 추천), SVG 생성 엔진 필수.
*   **데이터 구조 (Source of Truth):**
    *   `{word: string, meaning: string, style: string, svg_data: string, recommended_icons: []}`
    *   단순히 이미지를 저장하는 것이 아니라, **'단어-의미-SVG 속성'**을 포함하는 표준화된 데이터 구조가 필요합니다.
*   **최소 기능 구현 (MVP Scope):**
    *   단어 입력 및 픽토그램 검색/생성 (기본 기능).
    *   SVG/PNG 다운로드 (핵심 출력).
    *   *(선택적 기능: 사용자별 저장/로그인 등은 2단계 목표로 이월)*

### 📄 PICTO LAB MVP 구축을 위한 핵심 가이드라인 초안

사장님, 아래와 같이 3가지 핵심 영역을 중심으로 가이드라인을 초안화했습니다. 이 문서는 모든 에이전트의 작업 기준점이 됩니다.

#### 💡 1. 프로젝트 목표 및 사용자 경험 (UX/Business Focus)
*   **핵심 서비스:** 사용자가 단어(Text Input)만 입력하여 AI 기반의 전문 픽토그램을 생성/다운로드하는 솔루션.
*   **핵심 가치:** 건축/디자인 분야의 통일된 시각 언어(Visual Language) 제공.
*   **필수 플로우:**
    1.  **단어 입력:** 사용자가 키워드 입력.
    2.  **AI 추천/검색:** AI가 키워드에 맞는 픽토그램을 추천하거나 검색.
    3.  **스타일 적용:** 선형/면형/굵은 선형 등 정의된 스타일 시스템을 자동으로 적용하여 픽토그램 생성.
    4.  **다운로드:** SVG (벡터) 및 PNG (범용) 형태로 다운로드.

#### 🎨 2. 디자인 및 스타일 시스템 (Design Focus)
*   **스타일 원칙:** 모든 픽토그램은 통일된 시각 가이드라인을 준수해야 합니다.
    *   **선 굵기 (Stroke Weight):** 일관된 굵기 (예: 2pt) 사용.
    *   **모서리 처리:** 부드러운 곡률 (Radius) 적용.
    *   **구조:** 불필요한 디테일 제거, 최소한의 정보 전달에 집중 (Minimalism).
*   **스타일 시스템화:** 선형/면형/굵은 선형 등 여러 스타일 옵션을 제공하되, 각 스타일의 규칙을 명확히 정의하고 관리하는 **Style Sheet**가 필수적입니다.

#### 💻 3. 기술 및 데이터 구조 (Developer Focus)
*   **기술 스택:** 웹 기반 (React/Next.js 추천), SVG 생성 엔진 필수.
*   **데이터 구조 (Source of Truth):**
    *   `{word: string, meaning: string, style: string, svg_data: string, recommended_icons: []}`
    *   단순히 이미지를 저장하는 것이 아니라, **'단어-의미-SVG 속성'**을 포함하는 표준화된 데이터 구조가 필요합니다.
*   **최소 기능 구현 (MVP Scope):**

### 💡 Task 1: PICTO LAB 기술 명세서 최종 확정 (Tech Spec)
*   **목표:** 단어 입력 → AI 검색/추천 → SVG/PNG 출력까지의 모든 과정에 대한 기술적 흐름과 데이터 구조를 확정합니다.
*   **핵심 내용:**
    *   **데이터 구조:** '단어-의미-SVG 속성'의 표준화된 구조를 확정하고, 데이터베이스 스키마 초안을 작성합니다.
    *   **API 플로우:** 프론트엔드(입력)와 백엔드(AI 처리/SVG 생성) 간의 통신 프로토콜을 정의합니다.
*   **담당 에이전트:**
    *   **Developer:** 기술 아키텍처 담당 (Backend/API 플로우 설계)
    *   **Researcher:** 기술 난이도 분석 및 구현 가능성 검증
    *   **Designer:** 출력물(SVG)의 기술적 제약 사항 검토 및 반영

### 📐 Task 2: 픽토그램 스타일 시스템 가이드라인 확정 (Style Sheet)
*   **목표:** 통일성을 보장하는 시각적 규칙(선의 굵기, 곡률, 모서리 처리 등)을 측정 가능한 가이드라인으로 최종 확정합니다.
*   **핵심 내용:**
    *   **규칙 정의:** 모든 픽토그램이 따를 '선 가중치(Line Weight)', '곡률 반경(Curvature)', '모서리 처리 방식(Corner Radius)'을 수치적으로 확정합니다.
    *   **예외 처리:** 스타일 시스템을 위반할 수 있는 예외적인 상황(예: 복합 요소)에 대한 처리 규칙을 정의합니다.
*   **담당 에이전트:**
    *   **Designer:** 가이드라인의 시각적 완결성 검토 및 최종 초안 작성
    *   **Researcher:** 디자인 원칙에 대한 학술적/전문적 근거 제시
    *   **Writer:** Style Sheet 문서화 및 명료한 규칙서 작성

### 📈 Task 3: MVP 시장 검증 핵심 KPI 재확인 (Market Validation)
*   **목표:** MVP 출시 후, 서비스의 성공 여부를 측정할 핵심 지표(KPI)를 재확인하고, 검증을 위한 가설을 세웁니다.
*   **핵심 내용:**
    *   **핵심 지표:** '재방문율(Retention)', '특정 단어 검색 성공률(Search Success Rate)', 'SVG 다운로드 전환율' 등을 최우선 KPI로 확정합니다.
    *   **검증 플로우:** 어떤 사용자 그룹(Persona)에게, 어떤 시점에 KPI를 측정할지 시나리오를 정의합니다.
*   **담당 에이전트:**
    *   **Business:** 비즈니스 모델 관점에서의 수익화 및 지표 연계 검토
    *   **Writer:** KPI 정의의 명확화 및 사용자 관점의 가설 수립
    *   **Researcher:** 시장 데이터 및 경쟁사 분석 기반의 KPI 보강

# backend/main.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import List

app = FastAPI(title="AI Image Generator API")

# API 요청 본문 구조 정의
class QueryRequest(BaseModel):
    text_input: str
    style: str = "minimal"
    style_weights: List[float] = []

# API 응답 본문 구조 정의
class GenerationResponse(BaseModel):
    success: bool
    message: str
    generated_image_url: str
    metadata: dict

@app.post("/api/v1/generate", response_model=GenerationResponse)
async def generate_image(query: QueryRequest):
    """
    사용자가 입력한 텍스트와 스타일을 기반으로 가상의 이미지를 생성합니다.
    """
    print(f"Received request: Text='{query.text_input}', Style='{query.style}'")

    if not query.text_input:
        raise HTTPException(status_code=400, detail="텍스트 입력을 해주세요.")

    # --- 핵심 비즈니스 로직 시뮬레이션 ---
    # 실제로는 여기에 Stable Diffusion, DALL-E 등 외부 API 호출 로직이 들어갑니다.
    try:
        # 가상의 이미지 생성 성공 처리
        mock_url = f"https://mock-ai-service.com/images/{hash(query.text_input + query.style)}.png"
        return GenerationResponse(
            success=True,
            message=f"'{query.text_input}'에 대한 '{query.style}' 스타일 이미지가 성공적으로 생성되었습니다.",
            generated_image_url=mock_url,
            metadata={"input_text": query.text_input, "style": query.style}
        )
    except Exception as e:
        return GenerationResponse(
            success=False,
            message=f"이미지 생성 중 오류가 발생했습니다: {str(e)}",
            generated_image_url="",
            metadata={}
        )

@app.get("/health")
def read_root():
    return {"status": "ok", "service": "AI Image Generator API"}
```

```jsx
// frontend/src/App.jsx
import React, { useState } from 'react';
import './App.css';

function App() {
  const [prompt, setPrompt] = useState('');
  const [style, setStyle] = useState('minimal');
  const [result, setResult] = useState(null);
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (e) => {
    e.preventDefault();
    setLoading(true);
    setResult(null);

    try {
      const response = await fetch('http://localhost:8000/api/v1/generate', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          text_input: prompt,
          style: style,
          style_weights: []
        }),
      });

      const data = await response.json();
      setResult(data);

    } catch (error) {
      console.error("API 호출 오류:", error);
      setResult({ success: false, message: "네트워크 연결을 확인해주세요." });
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="App">
      <header>
        <h1>✨ AI 이미지 생성기</h1>
        <p>텍스트 프롬프트와 원하는 스타일을 입력하세요.</p>
      </header>

      <form onSubmit={handleSubmit} className="input-form">
        <input
          type="text"
          placeholder="생성하고 싶은 이미지에 대한 상세 설명 (예: 우주를 유영하는 고양이)"
          value={prompt}
          onChange={(e) => setPrompt(e.target.value)}
          disabled={loading}
        />
        <select value={style} onChange={(e) => setStyle(e.target.value)} disabled={loading}>
          <option value="minimal">미니멀리즘 (Minimal)</option>
          <option value="cinematic">시네마틱 (Cinematic)</option>
          <option value="watercolor">수채화 (Watercolor)</option>
        </select>
        <button type="submit" disabled={loading || !prompt}>
          {loading ? '생성 중...' : '이미지 생성하기'}
        </button>
      </form>

      {result && (
        <div className={`result-box ${result.success ? 'success' : 'error'}`}>
          <h2>{result.success ? '✅ 성공' : '❌ 실패'}</h2>
          <p>{result.message}</p>
          {result.success && (
            <div>
              <img src={result.generated_image_url} alt="Generated" style={{maxWidth: '100%', maxHeight: '300px', objectFit: 'contain', border: '1px solid #ccc'}} />
              <p>Meta: {JSON.stringify(result.metadata)}</p>
            </div>
          )}
        </div>
      )}
    </div>
  );
}

export default App;
```

```jsx
// frontend/src/App.jsx
import React, { useState } from 'react';
import './App.css';

function App() {
  const [prompt, setPrompt] = useState('');
  const [style, setStyle] = useState('minimal');
  const [result, setResult] = useState(null);
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (e) => {
    e.preventDefault();
    setLoading(true);
    setResult(null);

    try {
      const response = await fetch('http://localhost:8000/api/v1/generate', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          text_input: prompt,
          style: style,
          style_weights: []
        }),
      });

      const data = await response.json();
      setResult(data);

    } catch (error) {
      console.error("API 호출 오류:", error);
      setResult({ success: false, message: "네트워크 연결을 확인해주세요." });
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="App">
      <header>
        <h1>✨ AI 이미지 생성기</h1>
        <p>텍스트 프롬프트와 원하는 스타일을 입력하세요.</p>
      </header>

      <form onSubmit={handleSubmit} className="input-form">
        <input
          type="text"
          placeholder="생성하고 싶은 이미지에 대한 상세 설명 (예: 우주를 유영하는 고양이)"
          value={prompt}
          onChange={(e) => setPrompt(e.target.value)}
          disabled={loading}
        />
        <select value={style} onChange={(e) => setStyle(e.target.value)} disabled={loading}>
          <option value="minimal">미니멀리즘 (Minimal)</option>
          <option value="cinematic">시네마틱 (Cinematic)</option>
          <option value="watercolor">수채화 (Watercolor)</option>
        </select>
        <button type="submit" disabled={loading || !prompt}>
          {loading ? '생성 중...' : '이미지 생성하기'}
        </button>
      </form>

      {result && (
        <div className={`result-box ${result.success ? 'success' : 'error'}`}>
          <h2>{result.success ? '✅ 성공' : '❌ 실패'}</h2>
          <p>{result.message}</p>
          {result.success && (
            <div>
              <img src={result.generated_image_url} alt="Generated" style={{maxWidth: '100%', maxHeight: '300px', objectFit: 'contain', border: '1px solid #ccc'}} />
              <p>Meta: {JSON.stringify(result.metadata)}</p>
            </div>
          )}
        </div>
      )}
    </div>
  );
}

export default App;
